## Problem Statement

Stage 0 Gate Criterion #2 requires data contracts per layer. The Silver layer has no implementation yet, but the contract must define what transformations will be applied and what the output schema looks like before implementation begins.

---

## Context

Silver layer transforms Bronze (raw FHIR) into clinically meaningful, normalized data. Per IMPLEMENTATION_GUIDE.md, Silver must handle:
- Unit normalization (Celsius, mmHg, etc.)
- Time alignment to encounter timeline
- Missingness semantics
- Cohort definitions

This ticket defines the contract before Stage 2 implementation.

---

## Acceptance Criteria

- [ ] Contract document created (`contracts/silver_contract.md` or similar)
- [ ] Specifies input tables (from Bronze)
- [ ] Specifies output tables with schemas
- [ ] Documents unit normalization rules (which vitals, target units)
- [ ] Documents time alignment logic (anchor point, window definitions)
- [ ] Documents missingness handling strategy
- [ ] Documents cohort inclusion/exclusion criteria
- [ ] Explicitly states: "No model-specific logic in Silver layer"

---

## Approach

1. Review vital signs data in Bronze layer to understand source structure
2. Define target normalized schema for vital signs
3. Define encounter-aligned observation structure
4. Document cohort criteria (ED encounters with vital signs, valid disposition)
5. Create formal contract document

---

## Open Questions

- [ ] What is the anchor point for time alignment? (triage time? ED arrival? first vital?)
- [ ] What cohort exclusions? (e.g., exclude patients <18?)
- [ ] What missingness strategy? (carry-forward, null, imputation?)
