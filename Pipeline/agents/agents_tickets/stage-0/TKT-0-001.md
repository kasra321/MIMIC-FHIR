## Problem Statement

Stage 0 Gate Criterion #1 requires an architecture diagram. Currently no visual representation of the pipeline architecture exists. The IMPLEMENTATION_GUIDE.md has ASCII diagrams, but no formal architecture diagram suitable for technical review.

---

## Context

This is a Stage 0 blocking criterion. The diagram should formalize the medallion architecture and FHIR boundaries described in CLAUDE.md and IMPLEMENTATION_GUIDE.md.

---

## Acceptance Criteria

- [ ] Diagram shows FHIR input boundary (NDJSON sources)
- [ ] Diagram shows Bronze/Silver/Gold layer progression
- [ ] Diagram shows FHIR output boundary (RiskAssessment)
- [ ] Diagram indicates technology choices per layer (DuckDB, SQLMesh, etc.)
- [ ] Diagram saved in a format viewable in repo (SVG, PNG, or Mermaid in markdown)
- [ ] Diagram referenced from IMPLEMENTATION_GUIDE.md

---

## Approach

Create a Mermaid diagram embedded in a markdown file (`docs/architecture.md`) that can be rendered by GitHub and Obsidian. This keeps the diagram version-controlled and editable.

---

## Open Questions

- [ ] Should the diagram include the agent system or focus purely on data flow?
- [ ] What level of detail for technology stack (e.g., show DuckDB at Bronze, or abstract to "embedded database")?
