## Problem Statement

Stage 0 Gate Criterion #3 requires FHIR resource mapping to be complete. Currently, IMPLEMENTATION_GUIDE.md has tables describing ED-relevant profiles and LOINC codes, but no formal mapping document exists that:
- Maps MIMIC-IV FHIR profiles to pipeline tables
- Maps FHIR elements to specific columns needed
- Documents assumptions about FHIR structure

---

## Context

The MIMIC-IV on FHIR v2.1 documentation provides extensive detail about 30 profiles. For this project, we're using ED-specific profiles only. A formal mapping ensures we know exactly which FHIR paths we depend on.

**Current implementation uses:**
- MimicOrganization, MimicLocation, MimicPatient
- MimicEncounterED, MimicConditionED, MimicProcedureED
- MimicObservationED, MimicObservationVitalSignsED
- MimicMedicationDispenseED, MimicMedicationStatementED

---

## Acceptance Criteria

- [ ] Mapping document created (`contracts/fhir_mapping.md` or similar)
- [ ] Lists each MIMIC-IV FHIR profile used
- [ ] For each profile, documents:
  - FHIR paths required (e.g., `Observation.code.coding[0].code`)
  - Expected data types
  - Cardinality assumptions (single vs. array)
  - Any MIMIC-specific extensions used
- [ ] Documents vital sign LOINC codes expected
- [ ] Documents encounter disposition codes expected
- [ ] Cross-references assumptions in ASSUMPTIONS.md

---

## Approach

1. Sample each NDJSON file to understand actual structure
2. Document the FHIR paths currently accessed in `create_db.py` and `explore.ipynb`
3. Document the paths needed for Silver/Gold layer transformations
4. Flag any assumptions that need validation
